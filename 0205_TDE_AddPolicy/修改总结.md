# 修改汇总（0205 目录）

## 1. `0205/utils/temporal_dynamic_encoder_v2.py`
- 去除 reward 预测分支：
  - 删除 `reward_decoder`、`predict_reward`、reward 归一化与相关统计。
  - `compute_loss` 仅保留 `pred_loss`。
  - `update` 不再接收 `rewards`，返回值里移除 `tde_reward_loss` 等字段。
- 引入 AvgL1Norm（参考 TD7）：
  - 新增 `avg_l1_norm`，并在 `z_dynamic` 输出时应用。
- 新增 stop‑grad 预测接口：
  - `predict_next_from_z(z_dynamic, action)`：用于在 RL 前向中获取 `z_pred`（无梯度）。
- 精简统计与保存逻辑：
  - `stats` 只保留 `pred_loss_history` 与 `grad_norm_history`。
  - `save/load` 移除 reward 相关字段。

## 2. `0205/torchcore_true_dual.py`
- Policy/Actor 接口扩展：
  - `MLPGaussianPolicy.forward(x, z_dynamic=None)`，将 `z_dynamic` 拼入特征。
  - MLP 输入维度从 `136` 扩展为 `136 + tde_dim`。
- Critic 接口扩展：
  - Q 输入维度从 `128 + 8 + action` 扩展为 `128 + 8 + 2*tde_dim + action`。
- 新增 `actor()` / `critic()` 辅助函数：
  - `actor(x, z_dynamic)`：仅走策略网络。
  - `critic(x, a, z_dynamic, z_pred)`：走 Q 网络。
- `MLPActorCritic.forward` 兼容传入 `z_dynamic/z_pred`。

## 3. `0205/torchcore_true.py`
- 与 `torchcore_true_dual.py` 同步：
  - Policy 支持 `z_dynamic` 输入。
  - Critic 支持 `z_dynamic + z_pred`。
  - 新增 `actor()` / `critic()` 辅助函数。
  - 输入维度按 `tde_dim` 调整。

## 4. `0205/torchdclp_tde_fixed_test.py`
- 初始化更新：
  - `TDE_DIM = 64`，创建 `MLPActorCritic(..., tde_dim=TDE_DIM)`。
  - `TDEManagerV2` 初始化参数移除 reward 相关项。
- TDE 作为 stop‑grad 特征注入 Actor/Critic：
  - 新增 `get_tde_dynamic()` / `get_tde_pred()`。
  - `get_action()` 使用 `main_model.actor(..., z_dynamic)`。
  - Q 更新与 Policy 更新时使用 `z_dynamic` 和 `z_pred`。
- 日志精简：
  - TensorBoard 移除 `Loss/TDE_Reward` 与 reward 相关统计。

## 验证
- 已通过语法检查：
  - `python -m py_compile 0205/utils/temporal_dynamic_encoder_v2.py 0205/torchcore_true_dual.py 0205/torchcore_true.py 0205/torchdclp_tde_fixed_test.py`
